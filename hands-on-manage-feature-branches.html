Hands-On Activity: Collaborative Feature Development Using GitHub

Objective: In pairs, you'll each create and manage your own feature branches within a common GitHub project, focusing on adding or modifying HTML pages. This activity will teach you how to effectively use feature branches in a team setting.

Materials Needed: A computer with internet access and your preferred Git client (e.g., command line, GitHub Desktop).

Preparation:

  • Form Pairs: Partner up with a classmate. Although you'll work on individual feature branches, you'll collaborate and review each other's work.
  • Create an Account on The GitHub Homepage: If you haven't already, you must create an account on GitHub.

Step-by-Step Instructions:

In this collaborative exercise, you'll work in pairs to create, manage, and contribute to a shared GitHub repository. The process begins with one member of each pair setting up the repository and ends with both members contributing to it via individual feature branches.

Step 1: Create a New GitHub Repository (Performed by One Member)

Only one person in each pair needs to create the new GitHub repository. This simplifies the initial setup and ensures that both members work within the same project framework. The member who creates the repository will then share the repository URL with their partner, allowing both to clone the repository to their local machines.

  1. Log in to your GitHub account.
  2. Navigate to the Repositories tab and click the New button.
  3. Name your repository something descriptive related to this activity, e.g., web-project-activity.
  4. Choose Public (so your partner can see it) and initialize the repository with a README file.
  5. Click Create repository.

After creating the repository on GitHub, the member will add the basic HTML pages (index.html, about.html, contact.html) to the repository. This setup establishes the project structure that both members will contribute to.

Step 2: Clone Your New Repository

  • Using the Command Line:
    1. Open your terminal.
    2. Use git clone <your-new-repository-url> to clone the repository to your local machine.
  • Using GitHub Desktop:
    1. Open GitHub Desktop.
    2. Go to "File" > "Clone repository," select the URL tab, paste your new repository's URL, and choose where to save it locally.

Step 3: Add HTML Pages to Your Repository

Each student will add simple HTML pages to the repository. Below are templates for three basic HTML pages you can start with. Create each file in your local repository's root directory, add the HTML code provided, and save them with the suggested filenames.

1. Index Page (index.html): 

      <!DOCTYPE html>
<html>
<head>
    <title>My Web Project</title>
</head>
<body>
    <h1>Welcome to Our Web Project</h1>
    <p>This is a collaborative project by [Your Names].</p>
    <a href="about.html">About Us</a> | <a href="contact.html">Contact Us</a>
</body>
</html>

2. About Page (about.html): 

      <!DOCTYPE html>
<html>
<head>
    <title>About Us</title>
</head>
<body>
    <h1>About Us</h1>
    <p>We are students learning about web development and Git.</p>
    <a href="index.html">Home</a> | <a href="contact.html">Contact Us</a>
</body>
</html>

3. Contact Page (contact.html): 

      <!DOCTYPE html>
<html>
<head>
    <title>Contact Us</title>
</head>
<body>
    <h1>Contact Us</h1>
    <p>Email us at: student@example.com</p>
    <a href="index.html">Home</a> | <a href="about.html">About Us</a>
</body>
</html>

Step 4: Create Your Feature Branch

Each of you will create a branch for your specific feature.

  • Command Line: Run git checkout -b <your-feature-branch> in your project directory.
  • GitHub Desktop: Click on the "Current Branch" dropdown, then "New Branch" and name your branch.

Step 5: Add or Modify HTML Pages

On your branch, add a new HTML page or update an existing one. Ensure your HTML code is correct and check how it looks in a browser.

  • Example: If you're adding a "Contact Us" page, create a new HTML file named contact.html and write your HTML code.

Step 6: Commit Your Changes

Save your work in Git by committing your changes.

  • Command Line:
    1. git add . to stage all your changes.
    2. git commit -m "Your descriptive commit message" to commit.
  • GitHub Desktop:
    1. Check the changes you wish to stage.
    2. Write a commit message describing your changes and click "Commit".

Step 7: Push Your Branch to GitHub

Share your work by pushing your branch to the remote repository.

  • Command Line: Run git push -u origin <your-feature-branch>.
  • GitHub Desktop: Click "Publish branch" for the first push or "Push origin" for subsequent pushes.

Step 8: Review Your Partner's Work

After both of you have pushed your branches, take some time to review each other's changes.

  • Navigate to the GitHub repository online.
  • Switch between branches to view and discuss the changes.

Conclusion

Congratulations! You've successfully participated in a collaborative coding exercise using Git and GitHub. By creating feature branches, adding new content, and reviewing each other's work, you've practiced essential skills for team-based software development. This experience will serve as a foundation as you continue to explore more advanced Git operations and workflows.
